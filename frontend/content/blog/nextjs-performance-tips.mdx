## Overview

AI dashboards often combine dense tables, charting, streaming logs, and high-cardinality filters. The interface can feel slow even when the backend is healthy.

## Keep Data Boundaries Clear

Server components should fetch stable data like project metadata and reports. Client components should own volatile interactions such as filters, selected metrics, and lightbox state.

## Reduce Rendering Pressure

Virtualize large tables, memoize expensive transforms, and avoid placing every dashboard widget behind one giant client component.

## Measure the Real Flow

Performance work should focus on the path users repeat: opening the dashboard, filtering a model run, finding the anomaly, and exporting the evidence.
